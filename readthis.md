# PRD سیستم مدیریت پیشنهاد قیمت و محاسبه MTO باس‌داکت

## 1. معرفی محصول

این محصول یک سیستم نرم‌افزاری برای تعریف پروژه/پیشنهاد قیمت، انتخاب Scopeهای فنی، تعریف Lineهای پروژه، انتخاب قطعات مصرفی، محاسبه قیمت قطعات بر اساس جدول MTO و تولید خروجی نهایی قیمت است.

سیستم قرار است جایگزین یا مکمل فایل اکسل فعلی باشد که شامل شیت‌هایی مانند **All Update MTO**، **Material**، **Scope Name**، **Name / Parts**، **Line** و خروجی Pivot Table است.

هدف اصلی سیستم این است که کاربر بدون درگیر شدن با فرمول‌های اکسل، بتواند یک پروژه جدید تعریف کند، Scopeها و Lineهای آن را مشخص کند، قطعات هر Line را وارد کند و سیستم به‌صورت خودکار قیمت هر قطعه، هر Line، هر Scope و کل پروژه را محاسبه کند.

---

## 2. اهداف محصول

### اهداف اصلی

1. تبدیل فایل اکسل MTO به یک سیستم ساخت‌یافته و قابل توسعه.
2. تعریف پروژه‌ها یا پیشنهادهای قیمت جدید.
3. نگهداری جداول پایه شامل:

   * Material
   * Scope Name
   * Part Name
   * MTO Master
4. محاسبه خودکار قیمت قطعات بر اساس Scope و Part.
5. امکان تعریف قیمت متریال اختصاصی برای هر پروژه.
6. تولید خروجی نهایی مشابه Pivot Table فعلی.
7. آماده‌سازی سیستم برای توسعه‌های بعدی مانند سطح دسترسی کاربران، مس و آلومینیوم، گزارش‌گیری پیشرفته و نسخه‌بندی قیمت‌ها.

---

## 3. کاربران هدف

### 3.1 کاربر فروش / پیشنهاد قیمت

کاربری که پروژه‌ها یا پیشنهادهای قیمت را تعریف می‌کند، Scopeها را انتخاب می‌کند، Lineها را وارد می‌کند و خروجی قیمت را دریافت می‌کند.

### 3.2 مدیر یا کاربر ارشد

کاربری که احتمالاً در نسخه‌های بعدی بتواند همه پروژه‌ها را مشاهده کند، قیمت متریال پایه را مدیریت کند و دسترسی کاربران دیگر را کنترل کند.

### 3.3 کاربر فنی / مهندسی

کاربری که ساختار MTO، قطعات، متریال‌ها و ضرایب مصرف را تعریف یا اصلاح می‌کند.

* تعریف پروژه / پیشنهاد قیمت
* تعریف Scopeهای پروژه
* تعریف Lineهای هر Scope
* تعریف قطعات هر Line
* اتصال قطعات به جدول MTO
* محاسبه Unit Price و Total Price
* مدیریت قیمت متریال اختصاصی برای هر پروژه
* خروجی تجمیعی قیمت بر اساس Scope، Line و Part

### خارج از دامنه نسخه اول

* سطح دسترسی کامل کاربران
* مدیریت چندنقشی پیشرفته
* Workflow تأیید پیشنهاد
* ارسال خودکار پیشنهاد برای مشتری
* اتصال به ERP یا حسابداری

* طراحی خروجی رسمی PDF یا فرم تجاری نهایی، مگر به‌عنوان گزارش ساده

## 5. مفاهیم کلیدی سیستم

### 5.1 Project / Offer

هر پروژه یا پیشنهاد قیمت یک موجودیت مستقل است که کاربر ایجاد می‌کند.

فیلدهای پیشنهادی:

| فیلد                      | توضیح                                         |
| ------------------------- | --------------------------------------------- |
| Project ID                | شناسه یکتا                                    |
| Offer Name / Project Name | نام پیشنهاد یا پروژه                          |
| Offer Number              | شماره نامه یا شماره پیشنهاد                   |
| Type                      | نوع پروژه یا پیشنهاد                          |
| Input Date                | تاریخ ورود استعلام                            |
| Close Date                | آخرین مهلت ارائه پیشنهاد                      |
| Created By                | کاربر ایجادکننده                              |
| Created At                | تاریخ ایجاد                                   |
| Status                    | وضعیت پروژه، مانند Draft، In Progress، Closed |

---

### 5.2 Scope Name

Scope نشان‌دهنده مدل یا ظرفیت فنی است؛ مثلاً:

* 1000 AL
* 2500 AL
* 800 AL
* 6000 AL

در فایل فعلی، Scopeها در جدول مادر MTO و همچنین در شیت Material / Scope Name وجود دارند.

فیلدهای پیشنهادی:

| فیلد          | توضیح                    |
| ------------- | ------------------------ |
| Scope ID      | شناسه یکتا               |
| Scope Name    | نام Scope، مثلاً 1000 AL |
| Ampere        | مقدار آمپر               |
| Material Type | AL یا CU                 |
| Is Active     | فعال / غیرفعال           |

---

### 5.3 Part / Name

Part همان قطعه‌ای است که در فایل فعلی با فیلد **Name** مشخص شده است؛ مثلاً:

* فیدر ۱۰۰۰
* زانویی افقی
* زانویی عمودی
* شاخه مستقیم
* قطعات مرتبط با باس‌داکت

فیلدهای پیشنهادی:

| فیلد        | توضیح          |
| ----------- | -------------- |
| Part ID     | شناسه یکتا     |
| Part Name   | نام قطعه       |
| Description | توضیح اختیاری  |
| Is Active   | فعال / غیرفعال |

---

### 5.4 Material

Material جدول پایه متریال‌هاست. این جدول شامل نام متریال، قیمت پیش‌فرض، واحد و ابعاد است.

فیلدهای پیشنهادی:

| فیلد               | توضیح                         |
| ------------------ | ----------------------------- |
| Material ID        | شناسه یکتا                    |
| Material Name      | نام متریال                    |
| Default Unit Price | قیمت پیش‌فرض                  |
| Unit               | واحد، مثلاً کیلوگرم، متر، عدد |
| Dimension          | ابعاد                         |
| Is Active          | فعال / غیرفعال                |

---

### 5.5 MTO Master

جدول MTO Master همان جدول مادر فعلی، یعنی **All Update MTO** است. این جدول مشخص می‌کند هر Part در هر Scope از چه متریال‌هایی و با چه ضریب مصرفی استفاده می‌کند.

فیلدهای مهم از فایل فعلی:

| فیلد        | توضیح                                                          |
| ----------- | -------------------------------------------------------------- |
| Scope Name  | Scope مربوطه، مثلاً 1000 AL                                    |
| Name        | نام قطعه / Part                                                |
| Material    | متریال مصرفی                                                   |
| Quantity    | تعداد، فعلاً نگهداری می‌شود اما در محاسبه اصلی استفاده نمی‌شود |
| Value       | ضریب مصرف، فیلد اصلی برای محاسبه                               |
| Unit        | واحد                                                           |
| Unit Price  | قیمت واحد متریال                                               |
| Total Price | قیمت کل، حاصل Value × Unit Price                               |

نکته مهم: در نسخه اول، **Value فیلد اصلی محاسباتی است**. Quantity ذخیره می‌شود اما فعلاً مبنای اصلی محاسبه نیست.

---

### 5.6 Project Material Price

برای هر پروژه، سیستم باید یک نسخه اختصاصی از قیمت متریال‌ها داشته باشد.

زمانی که پروژه ایجاد می‌شود، سیستم باید از جدول Material پایه یک کپی برای همان پروژه بسازد. این کار باعث می‌شود قیمت متریال هر پروژه مستقل باشد و تغییر قیمت در یک پروژه روی پروژه‌های دیگر اثر نگذارد.

فیلدهای پیشنهادی:

| فیلد                      | توضیح                     |
| ------------------------- | ------------------------- |
| Project Material Price ID | شناسه یکتا                |
| Project ID                | شناسه پروژه               |
| Material ID               | شناسه متریال              |
| Unit Price                | قیمت واحد مخصوص این پروژه |
| Unit                      | واحد                      |
| Dimension                 | ابعاد                     |
| Created At                | تاریخ ایجاد               |

---

### 5.7 Line

Line مسیر یا بخش اجرایی در پروژه است. مشابه مسیر لوله‌کشی یا گازکشی، هر Line می‌تواند شامل چند قطعه باشد.

مثال:

* Scope: 1000 AL

  * Line 1

    * فیدر ۱۰۰۰، تعداد ۲
    * زانویی افقی، تعداد ۱
  * Line 2

    * شاخه مستقیم، تعداد ۴
    * زانویی عمودی، تعداد ۲

فیلدهای پیشنهادی Line:

| فیلد        | توضیح                  |
| ----------- | ---------------------- |
| Line ID     | شناسه یکتا             |
| Project ID  | شناسه پروژه            |
| Scope ID    | Scope مربوط به Line    |
| Line Name   | نام Line، مثلاً Line 1 |
| Description | توضیح اختیاری          |

---

### 5.8 Line Item

هر Line شامل تعدادی قطعه است. Line Item مشخص می‌کند در یک Line چه قطعه‌ای و به چه تعداد استفاده شده است.

فیلدهای پیشنهادی:

| فیلد         | توضیح                             |
| ------------ | --------------------------------- |
| Line Item ID | شناسه یکتا                        |
| Line ID      | شناسه Line                        |
| Part ID      | شناسه قطعه                        |
| Quantity     | تعداد قطعه در این Line            |
| Unit Price   | قیمت واحد قطعه، محاسبه‌شده از MTO |
| Total Price  | حاصل Quantity × Unit Price        |

---

## 6. منطق محاسبات

### 6.1 محاسبه قیمت یک Part در یک Scope

برای محاسبه قیمت یک قطعه در یک Scope، سیستم باید در جدول MTO Master جست‌وجو کند و ردیف‌هایی را پیدا کند که:

* Scope Name برابر Scope انتخاب‌شده باشد.
* Name / Part برابر قطعه انتخاب‌شده باشد.

سپس برای هر ردیف:

```text
Material Total Price = Value × Project Material Unit Price
```

در نهایت:

```text
Part Unit Price = Sum(Material Total Price for selected Scope and Part)
```

یعنی مشابه فرمول SumIf / SumIfs فعلی در اکسل.

---

### 6.2 محاسبه قیمت قطعه در Line

پس از به‌دست‌آمدن Unit Price قطعه:

```text
Line Item Total Price = Part Unit Price × Quantity
```

---

### 6.3 محاسبه قیمت Line

```text
Line Total Price = Sum(Line Item Total Price)
```

---

### 6.4 محاسبه قیمت Scope

```text
Scope Total Price = Sum(Line Total Price for all Lines under Scope)
```

---

### 6.5 محاسبه قیمت کل پروژه

```text
Project Total Price = Sum(Scope Total Price)
```

---

## 7. جریان کار کاربر

### 7.1 ایجاد پروژه جدید

کاربر وارد صفحه پروژه‌ها می‌شود و روی گزینه **New Offer** یا **New Project** کلیک می‌کند.

کاربر فیلدهای زیر را وارد می‌کند:

* نام پروژه / پیشنهاد
* شماره پیشنهاد یا نامه
* نوع پروژه
* تاریخ ورود
* تاریخ مهلت تحویل / Close Date

پس از ثبت پروژه، سیستم قیمت متریال پایه را برای پروژه کپی می‌کند و Project Material Price ایجاد می‌شود.

---

### 7.2 تعریف Scopeهای پروژه

کاربر در پروژه جدید، Scopeهای موردنیاز را انتخاب می‌کند.

مثال:

* 1000 AL
* 2500 AL

هر پروژه می‌تواند چند Scope داشته باشد.

---

### 7.3 تعریف Line برای هر Scope

برای هر Scope، کاربر می‌تواند چند Line تعریف کند.

مثال:

* 1000 AL

  * Line 1
  * Line 2
  * Line 3

---

### 7.4 افزودن قطعات به Line

در هر Line، کاربر قطعات موردنیاز را انتخاب و تعداد آن‌ها را وارد می‌کند.

مثال:

| Line   | Part        | Quantity |
| ------ | ----------- | -------- |
| Line 1 | فیدر ۱۰۰۰   | ۲        |
| Line 1 | زانویی افقی | ۱        |
| Line 2 | شاخه مستقیم | ۴        |

کاربر نیازی ندارد قیمت را دستی وارد کند. سیستم قیمت را از MTO Master و قیمت متریال همان پروژه محاسبه می‌کند.

---

### 7.5 مشاهده خروجی قیمت

سیستم خروجی‌ای مشابه Pivot Table فعلی نمایش می‌دهد:

| Scope   | Line   | Part        | Quantity | Unit Price | Total Price |
| ------- | ------ | ----------- | -------: | ---------: | ----------: |
| 1000 AL | Line 1 | فیدر ۱۰۰۰   |        ۲ | محاسبه‌شده |  محاسبه‌شده |
| 1000 AL | Line 1 | زانویی افقی |        ۱ | محاسبه‌شده |  محاسبه‌شده |
| 1000 AL | Line 2 | شاخه مستقیم |        ۴ | محاسبه‌شده |  محاسبه‌شده |

در انتها جمع‌ها باید قابل مشاهده باشند:

* جمع هر Line
* جمع هر Scope
* جمع کل پروژه

---

## 8. نیازمندی‌های عملکردی

### FR-001: مدیریت پروژه‌ها

سیستم باید امکان مشاهده لیست پروژه‌ها را فراهم کند.

هر ردیف پروژه باید حداقل شامل موارد زیر باشد:

* نام پروژه
* شماره پیشنهاد
* تاریخ ورود
* تاریخ بسته‌شدن
* وضعیت
* کاربر ایجادکننده
* مبلغ کل، در صورت محاسبه

---

### FR-002: ایجاد پروژه جدید

سیستم باید امکان ایجاد پروژه جدید از طریق دکمه **New Offer / New Project** را فراهم کند.

پس از ایجاد پروژه، سیستم باید به‌صورت خودکار قیمت‌های پیش‌فرض Material را برای آن پروژه کپی کند.

---

### FR-003: مدیریت Material پایه

سیستم باید امکان تعریف، ویرایش و غیرفعال‌کردن Material را فراهم کند.

هر Material باید شامل موارد زیر باشد:

* نام متریال
* قیمت پیش‌فرض
* Unit
* Dimension

---

### FR-004: مدیریت قیمت متریال پروژه

سیستم باید برای هر پروژه جدول قیمت متریال اختصاصی داشته باشد.

کاربر باید بتواند قیمت متریال‌ها را در سطح پروژه تغییر دهد بدون اینکه قیمت پایه یا پروژه‌های دیگر تغییر کند.

---

### FR-005: مدیریت Scope

سیستم باید امکان تعریف Scopeهای پایه را داشته باشد.

در نسخه اول، Scopeهای موجود از اکسل فعلی وارد سیستم می‌شوند.

---

### FR-006: مدیریت Part

سیستم باید امکان تعریف قطعات پایه را داشته باشد.

Partها باید در Line Item قابل انتخاب باشند.

---

### FR-007: مدیریت MTO Master

سیستم باید جدول MTO Master را نگهداری کند.

هر رکورد MTO باید مشخص کند:

* برای چه Scopeای است.
* مربوط به چه Partای است.
* از چه Materialای استفاده می‌کند.
* Value / ضریب مصرف آن چقدر است.
* Quantity آن چقدر است، حتی اگر فعلاً در محاسبه استفاده نشود.
* Unit آن چیست.

---

### FR-008: تعریف Scope برای پروژه

در هر پروژه، کاربر باید بتواند یک یا چند Scope انتخاب کند.

---

### FR-009: تعریف Line برای Scope

کاربر باید بتواند برای هر Scope چند Line تعریف کند.

هر Line باید زیرمجموعه یک Scope و یک پروژه باشد.

---

### FR-010: تعریف قطعات Line

کاربر باید بتواند برای هر Line، قطعات موردنیاز و تعداد آن‌ها را وارد کند.

---

### FR-011: محاسبه Unit Price قطعه

سیستم باید Unit Price هر Part را بر اساس Scope انتخاب‌شده، Part انتخاب‌شده و Project Material Price محاسبه کند.

---

### FR-012: محاسبه Total Price قطعه

سیستم باید Total Price هر Line Item را با فرمول زیر محاسبه کند:

```text
Total Price = Quantity × Unit Price
```

---

### FR-013: خروجی تجمیعی

سیستم باید خروجی قیمت را به تفکیک موارد زیر نمایش دهد:

* Project
* Scope
* Line
* Part
* Quantity
* Unit Price
* Total Price

---

### FR-014: جمع نهایی

سیستم باید جمع قیمت را در سطوح زیر نمایش دهد:

* جمع هر Line
* جمع هر Scope
* جمع کل Project / Offer

---

## 9. نیازمندی‌های غیرفعملکردی

### 9.1 دقت محاسبات

محاسبات قیمت باید دقیق و قابل ردیابی باشد. هر قیمت نهایی باید قابل برگشت به رکوردهای MTO و قیمت متریال پروژه باشد.

### 9.2 توسعه‌پذیری

سیستم باید به‌گونه‌ای طراحی شود که در آینده بتوان موارد زیر را اضافه کرد:

* CU / مس
* سطح دسترسی کاربران
* نسخه‌بندی پیشنهادها
* خروجی PDF
* فرمول‌های قیمت‌گذاری پیچیده‌تر
* تأیید مدیریتی

### 9.3 قابلیت نگهداری

دیتاهای پایه مانند Material، Scope، Part و MTO نباید به‌صورت hard-code در برنامه باشند و باید از طریق دیتابیس مدیریت شوند.

### 9.4 کاربری ساده

کاربر فروش یا اپراتور نباید با فرمول‌های محاسبه قیمت درگیر شود. او فقط باید Scope، Line، Part و Quantity را وارد کند.

## 12. قوانین تجاری

1. هر پروژه باید حداقل یک Scope داشته باشد.
2. هر Scope در پروژه می‌تواند چند Line داشته باشد.
3. هر Line می‌تواند چند Part داشته باشد.
4. قیمت متریال هر پروژه مستقل از پروژه‌های دیگر است.
5. هنگام ایجاد پروژه، قیمت‌های Material پایه باید در Project Material Price کپی شوند.
6. Unit Price قطعه نباید به‌صورت دستی توسط کاربر وارد شود، بلکه باید از MTO و قیمت متریال پروژه محاسبه شود.
7. Quantity در MTO فعلاً ذخیره می‌شود اما مبنای اصلی محاسبه نیست.
8. Value در MTO ضریب مصرف اصلی است.
9. اگر برای ترکیب Scope و Part در MTO رکوردی وجود نداشت، سیستم باید خطا یا هشدار نمایش دهد.
10. اگر قیمت متریال پروژه تعریف نشده بود، سیستم باید از محاسبه جلوگیری کند یا هشدار دهد.

## 13. سناریو نمونه

### ورودی

کاربر پروژه‌ای با نام «پروژه الف» ایجاد می‌کند.

اطلاعات پروژه:

* Offer Number: OF-001
* Input Date: 2026-05-07
* Close Date: 2026-05-15

کاربر Scopeهای زیر را انتخاب می‌کند:

* 1000 AL

برای Scope 1000 AL، دو Line تعریف می‌کند:

* Line 1
* Line 2

برای Line 1 قطعات زیر را وارد می‌کند:

| Part        | Quantity |
| ----------- | -------: |
| فیدر ۱۰۰۰   |        ۲ |
| زانویی افقی |        ۱ |

سیستم برای Part «فیدر ۱۰۰۰» در Scope «1000 AL» به جدول MTO مراجعه می‌کند، متریال‌های مرتبط را پیدا می‌کند، Value هر متریال را در Unit Price همان متریال برای پروژه ضرب می‌کند و جمع آن را به‌عنوان Unit Price قطعه نمایش می‌دهد.

سپس:

```text
Total Price = Unit Price × Quantity
```

در نهایت قیمت Line، Scope و کل پروژه محاسبه می‌شود.

| Scope   | Line   | Part        | Quantity | Unit Price | Total Price |
| ------- | ------ | ----------- | -------: | ---------: | ----------: |
| 1000 AL | Line 1 | فیدر ۱۰۰۰   |        ۲ | محاسبه‌شده |  محاسبه‌شده |
| 1000 AL | Line 1 | زانویی افقی |        ۱ | محاسبه‌شده |  محاسبه‌شده |

| سطح           |         مبلغ |
| ------------- | -----------: |

## 15. موارد ابهام / نیازمند تصمیم

1. آیا Quantity موجود در MTO در آینده وارد فرمول محاسبه می‌شود یا فقط برای نمایش است؟
2. آیا کاربر باید بتواند Unit Price قطعه را Override کند؟
3. آیا برای هر پروژه چند نسخه قیمت یا Revision لازم است؟
4. آیا خروجی رسمی برای مشتری لازم است یا فقط خروجی داخلی کافی است؟
5. آیا CU / مس در نسخه اول باید پشتیبانی شود یا بعداً اضافه می‌شود؟
6. آیا Scopeها فقط از لیست ثابت انتخاب می‌شوند یا کاربر می‌تواند Scope جدید بسازد؟
7. آیا Partها فقط از لیست پایه انتخاب می‌شوند یا در سطح پروژه قابل تعریف هستند؟
8. سطح دسترسی کاربران در نسخه اول حذف می‌شود یا حداقل باید Created By لحاظ شود؟
9. آیا نیاز به Import مستقیم از فایل Excel وجود دارد؟
10. آیا خروجی باید Excel هم داشته باشد؟

## 16. معیارهای پذیرش

### AC-001

کاربر بتواند پروژه جدید با نام، شماره پیشنهاد، تاریخ ورود و تاریخ بسته‌شدن ایجاد کند.

### AC-002

پس از ایجاد پروژه، سیستم به‌صورت خودکار Material Price اختصاصی پروژه را از جدول Material پایه بسازد.

### AC-004

### AC-005

کاربر بتواند برای هر Line چند Part و Quantity وارد کند.

### AC-006

سیستم بتواند برای هر Part، بر اساس Scope و MTO Master، Unit Price را محاسبه کند.

سیستم بتواند Total Price هر Part را بر اساس Quantity محاسبه کند.

### AC-010
