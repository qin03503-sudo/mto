# MTO Pricing Web App Documentation

This directory contains the structured project documentation used by product, frontend, backend, and QA teams during development.

Start here:

1. Read `../proposal.md` for the complete product proposal generated from `readthis.md`.
2. Read `00_Project_Overview/readme.md` to understand the project goal.
3. Read `00_Project_Overview/glossary.md` to align on domain terms.
4. Use `01_Product_Requirements_PRD/` for product scope and user flows.
5. Use `02_Architecture_System_Design/`, `03_Database_Design/`, `04_API_Contracts/`, and `06_Calculation_Engine/` for implementation planning.
6. Use `08_Testing_and_QA/` to validate calculation accuracy against the Excel source.

The source notes used to start this structure are in the repository root: `start.md` and `unstrucured.md`.
